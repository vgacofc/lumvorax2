Le texte est structuré pour être exécuté directement par un agent autonome.

Prompt Agent — Audit et Correction Globale des Normalisations

Mission

Analyser l’intégralité du dépôt scientifique, détecter toutes les normalisations implicites ou artificielles, puis corriger les modules afin que :
	•	les sorties proxy
	•	les benchmarks externes

soient exprimés dans les mêmes unités physiques et le même format numérique.

Aucune correction par facteur arbitraire (ex: 1e6) ne doit subsister.

4. Correction automatique des modules physiques

Appliquer les corrections suivantes.

4.1 Hubbard HTS

Fichiers typiques :

hubbard_hts_core/
hubbard_hts_research_cycle.c

Correction obligatoire :

energy_total = compute_energy(state)
energy_per_site = energy_total / lattice_size

Supprimer tout facteur constant.

4.2 QCD lattice proxy

Vérifier normalisation :

action / volume
plaquette averages

Sorties requises :

action_total
action_per_site

4.3 Dense nuclear proxy

Contrôler :

energy density
baryon density
pressure

Sorties standardisées :

energy_total
energy_per_particle
energy_density

4.4 Quantum chemistry proxy

Vérifier conversion entre :

Hartree
eV
kJ/mol

Sortie obligatoire :

energy_hartree
energy_ev
energy_kjmol

5. Format universel des sorties

Tous les modules doivent produire le même schéma JSON.

Exemple :

{
  "module": "hubbard_hts",
  "lattice_size": 4096,
  "parameters": {
    "U": 8.0,
    "t": 1.0
  },
  "energy_total": -10324.554382919,
  "energy_per_site": -2.520641384912,
  "benchmark_energy_per_site": -2.519800000000,
  "relative_error": 0.000333,
  "within_error_bar": true
}

6. Format numérique obligatoire

Toutes les sorties doivent utiliser :

%.12f

Exemple :

-2.520641384912

Interdits :

≈
~
1e6
1e-3

Les valeurs doivent être complètes et explicites.
